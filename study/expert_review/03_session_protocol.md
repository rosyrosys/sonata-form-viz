# Session Protocol — Researcher's 1-Hour Script

Read this before each session. The script is a *guide*, not a
rigid teleprompter — adapt to the expert's tempo and conversational
style.

**Total duration:** 60 minutes (5 + 10 + 30 + 5 + 15 = 65 min,
allow 5 min buffer)
**Required:** 도구 URL, 인터뷰 가이드 (file 05), SUS 양식 (file 04),
녹음 도구 (스마트폰 or Zoom recording), 노트 양식 (file 06)

---

## Pre-session checklist (do before each session)

- [ ] 참여자 안내서 (file 02) 미리 송부했는가
- [ ] 참여자가 회신으로 동의 항목 확인했는가
- [ ] 도구 URL 공유 가능 상태인가 (https://rosyrosys.github.io/sonata-form-viz/)
- [ ] 녹음 도구 작동 확인 (Zoom 의 경우 cloud recording 켜기)
- [ ] 인터뷰 가이드 + SUS 양식 한 화면에 띄워두기
- [ ] 노트 양식 (file 06 의 사본) 준비

---

## 0–5 분 — 도입

**스크립트:**
> "안녕하세요 [성명] 선생님, 오늘 시간 내주셔서 감사합니다. 안내서
> 미리 보내드린 대로 약 1시간 진행됩니다. 시작 전에 두 가지 확인
> 하겠습니다. 첫째, 본 세션 녹음에 동의하시는지요? 둘째, 결과는
> Expert A 또는 Expert B 로 익명 보고되며, acknowledgments 에
> 성명 표기는 별도 동의 사항입니다."

- [ ] 녹음 동의 확인 → 녹음 시작
- [ ] Acknowledgments 표기 동의 확인 → 노트 양식에 기록

**스크립트 (계속):**
> "본 도구는 모차르트 K. 545 1악장의 악보·음원·형식 분석을 동기화한
> 학습자용 시각화 도구입니다. 오늘은 세 단계로 진행합니다: 먼저
> 10분간 자유롭게 도구를 살펴보시고, 다음 30분간 원고 §5 의 30분
> 교실 워크-스루를 *선생님이 학부생을 가르친다고 가정* 하고 진행해
> 주세요. 마지막에 5분 사용성 척도와 15분 인터뷰입니다."

- [ ] 도구 URL 공유: https://rosyrosys.github.io/sonata-form-viz/

---

## 5–15 분 — 자율 탐색 (10분)

**연구자 행동:**
- 침묵하며 관찰. 참여자가 자발적으로 사고-구술 (think-aloud) 하면
  격려, 안 하면 그대로 두기.
- 노트: 첫 클릭 위치, 처음 발견한 기능, 헷갈리는 표현 등.

**개입 시점:**
- 참여자가 5분 이상 같은 화면에 머물면: "어떤 부분이 궁금하세요?"
- 참여자가 10분 채우기 전에 "더 볼 게 없는 것 같다" 하면: 다음
  단계로 진행.

**기록할 것 (노트 양식 file 06 → "자율 탐색 관찰"):**
- 첫 3분에 시도한 기능 목록
- 도구의 어느 요소를 *바로 이해* 했는가 (색상? 타임라인? 점프 버튼?)
- 어느 요소가 *발견되지 않았는가* (T-키 보정? 데모 모드?)

---

## 15–45 분 — 30분 교실 워크-스루 시연 (30분)

**스크립트:**
> "이제 원고의 §5 워크-스루를 선생님이 학부 음악교육 전공자에게
> 가르친다고 가정하고 진행해 주세요. 5단계로 구성되어 있습니다:"

워크-스루 5단계 안내 (참여자가 화면에 띄워두도록 file 06 발췌 송부
또는 화면 공유):

1. **첫 청취 (5분)** — 색상 끈 상태로 전악장 청취. "악장이 어디서
   나뉜다고 생각하나요?" 라는 질문을 학생들에게 던지는 시뮬레이션.
2. **색상 공개와 섹션 식별 (8분)** — Section color 토글 후 학생들이
   섹션 경계를 식별하는 단계.
3. **제시부 P 와 S (5분)** — 섹션 점프로 P, S 만 재생. 학생에게
   화성 진행 질문.
4. **재현부 P (8분)** — *결정적 순간*. 재현부 점프, "F major" 라벨
   확인, A/B 비교 (제시부 P vs 재현부 P).
5. **종합 (4분)** — 도식 비교 (C→G / F→C), "왜 모차르트는 '틀린'
   조성에서 재현부를 시작했나" 토론 유도.

**연구자 행동:**
- 참여자가 *학생 역할* 을 가상의 학생에게 묻는다고 상상하며 진행하게
  격려.
- 각 단계에서 *관찰한 것* 을 노트 양식의 해당 칸에 기록:
  - 단계별 막힘 점 (e.g., 색상 토글 위치를 못 찾음)
  - 단계별 "학생이라면 헷갈렸을 것" 코멘트
  - 단계별 *추가하면 좋겠다는* 기능 제안

**시간 관리:**
- 각 단계 종료 시점에 부드럽게 다음 단계로 안내 ("좋습니다,
  이제 다음 단계로 가시죠")
- 30분이 다 되면 종료 신호: "5단계까지 마치셨네요, 이제 사용성
  척도로 넘어가겠습니다"

---

## 45–50 분 — SUS 양식 (5분)

**스크립트:**
> "5분 동안 시스템 사용성 척도 10문항에 응답해 주세요. 각 문항에
> '매우 동의 안 함 (1점) ~ 매우 동의 (5점)' 으로 표시하시면 됩니다.
> 정답은 없습니다, 첫 직관으로 응답해 주세요."

- [ ] SUS 양식 송부 (file 04 의 Google Forms 링크 or PDF or 화면
      공유 양식)
- 침묵하며 참여자 작성 대기
- 작성 완료 응답 받기

**참여자 친절한 안내:**
> "혹시 두 답 사이에서 망설이시면, 도구 *전체* 의 인상으로
> 응답해 주세요. 특정 기능 하나만 떠올리지 마시고요."

---

## 50–65 분 — 반-구조 인터뷰 (15분)

**file 05 의 인터뷰 가이드 사용.** 6개 핵심 질문 + 후속 질문.

**연구자 행동:**
- 핵심 질문은 *문구 그대로* 묻기 (질적 분석 일관성 위해)
- 후속 질문 ("좀 더 자세히 말씀해 주시면..."/ "예를 들면?") 자유롭게
- 답변이 풍부하면 시간 초과 허용 (75분까지)
- 답변이 짧으면 다음 질문으로

**기록할 것:**
- 인용 가능한 *발화 그대로* (워드 단위)
- 비언어적 신호 (망설임, 강조, 감탄)
- 후속 RCT 에 반영할 만한 *설계 권고* 모두

---

## 종료 (1분)

**스크립트:**
> "오늘 정말 감사합니다. 결과는 원고 §6 으로 정리되며, 게재 후
> Zenodo 영구 보관 링크 보내드리겠습니다. 추가로 떠오르시는 점이
> 있으면 일주일 안에 이메일로 보내주셔도 좋습니다."

- [ ] 녹음 종료
- [ ] 노트 양식 (file 06) 완성도 확인 (자율 탐색·워크-스루·SUS·
      인터뷰 4개 섹션 모두 작성되었는가)
- [ ] 즉시 추가 기억 노트 (post-session memo): 인상 깊은 코멘트,
      예상 못한 발견 등

---

## After both sessions

- [ ] 녹음 → 전사 (Whisper 또는 수동, 1시간 ≈ 4시간 전사 시간)
- [ ] SUS 점수 계산 (file 04 의 채점 규칙)
- [ ] 인터뷰 코딩 (file 06 의 thematic analysis 워크시트)
- [ ] §6 초고 작성 (file 06 의 paper 템플릿)
